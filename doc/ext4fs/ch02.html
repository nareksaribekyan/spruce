Chapter 2. How to execute the module

It is not recommended to run the Ext4 FS module alone, but if you know what are you doing then here are some hints which can help you to execute the module in several scenarios.

The module execution can be controlled by it's configuration file (ext4fs.conf in config directory). It consists of several lines each describing a single test. Lines must have the following format: OPERATION_IDENTIFIER ARG1 ARG2 ... ARGN. All the possible operation identifiers are listed below.

  • Ext4IoctlSetFlagsGetFlags - Try to set and then get some flag value.
  • Ext4IoctlClearExtentsFlags - Try to clear the Extents flag, which is not allowed!
  • Ext4IoctlSetFlagsNotOwner - Try to set flags not having permission to that (not the file owner or have no capabilities.
  • Ext4IoctlSetVersionGetVersion - Try to set and then get the version.
  • Ext4IoctlGroupExtend - Online resize test.
  • Ext4IoctlMoveExtent - Online defragmentation test.
  • Ext4IoctlAllocDABlocks - Delayed allocation test.
  • Ext4IoctlUnsupported - Try to execute an unsupported command
